# apps/labs/components

This directory contains the specialized UI component library for the `apps/labs` ecosystem within the ekkOS platform. It provides modular, React-based components designed to render research-oriented interfaces, analytical dashboards, and documentation-style layouts that adhere to the platform's "cyber" aesthetic.

## Architecture
- **Runtime**: React (TypeScript/TSX)
- **Entry points**: 
    - `scorecard/ScorecardDashboard.tsx`: Primary entry for analytical data visualization.
    - `ResearchCard.tsx`, `CodeWindow.tsx`, `RoadmapSection.tsx`: Core functional components for content rendering.
- **Key dependencies**: 
    - `packages-ui`: Foundational design system and primitives.
    - `apps-labs-lib`: Data hooks and business logic providers.
    - `react`: UI rendering engine.
    - `typescript`: Type safety for component props.
    - `tailwind-css` (inferred): Styling utility for the "cyber" aesthetic.
- **Build system**: Managed by the monorepo root (Turborepo/Nx inferred).
- **Deploy target**: Client-side browser environment via `apps/labs`.

## Key Files
- **`scorecard/ScorecardDashboard.tsx`**: Renders complex analytical metrics and research data.
- **`ResearchCard.tsx`**: Container for presenting research findings or data snippets.
- **`CodeWindow.tsx`**: Specialized component for rendering technical code blocks.
- **`CyberBackground.tsx`**: Provides the consistent "cyber" visual branding.
- **`RoadmapSection.tsx`**: Layout component for project milestones and timelines.
- **`PageContainer.tsx`**: Wrapper for consistent layout constraints.
- **`EkkosLogo.tsx`**: Centralized branding component.
- **`Badge.tsx` & `CTAButton.tsx`**: Atomic UI elements for status and interaction.
- **`Section.tsx`**: Generic layout wrapper for content segmentation.
- **`QuoteBlock.tsx`**: Styled component for highlighting key insights.

## API / Exports
- **`ScorecardDashboard`**: Accepts data props for rendering metrics; integrates with `apps-labs-lib` hooks.
- **`ResearchCard`**: Accepts `title`, `content`, and `metadata` props.
- **`CodeWindow`**: Accepts `code` string and `language` identifier.
- **`CTAButton`**: Standardized button interface for platform actions.

## Environment Variables
*No explicit environment variables are defined in the component files.* These components are designed to be consumed by `apps-labs-app`, which injects configuration via context providers or `lib-config`.

## Conventions
- **Component Pattern**: Functional React components using TypeScript interfaces for props.
- **Styling**: Tailwind CSS classes are used for layout and "cyber" aesthetic styling.
- **Organization**: Components are grouped by function; specialized dashboards (like `scorecard/`) reside in subdirectories.
- **Error Handling**: Components should be wrapped in Error Boundaries provided by the parent application.
- **Logging**: Minimal local logging; relies on platform-wide observability tools.
