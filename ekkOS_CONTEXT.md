# apps/labs

`apps/labs` is the official research and innovation portal for ekkOS Technologies Inc. It serves as a public-facing Next.js application that hosts research articles, technical roadmaps, and high-fidelity interactive engineering demonstrations, providing transparency into the company's ongoing R&D initiatives.

## Architecture
- **Runtime**: Node.js (Next.js 16.0.10, App Router)
- **Entry points**: `app/layout.tsx` (root layout), `app/page.tsx` (home), `app/research/page.tsx` (research index)
- **Key dependencies**:
  - `@ekkos/gemini`: Internal package for AI/LLM integration.
  - `@upstash/redis`: Serverless Redis for state management and caching.
  - `@supabase/supabase-js`: Database and authentication client.
  - `next`: Framework core.
  - `lucide-react`: Iconography.
  - `tailwind-merge` & `clsx`: Styling utilities.
- **Build system**: `next build` (configured for `standalone` output).
- **Deploy target**: Vercel (region `iad1`).

## Key Files
- `app/research/[slug]/page.tsx`: Dynamic route for rendering research articles.
- `lib/research.ts`: Central data store for research article metadata and content.
- `app/engineering/`: Directory containing high-fidelity 3D/interactive components (e.g., `LayerStack3D.tsx`, `WireframeGlobe.tsx`).
- `app/api/ekk0/chat/route.ts`: API endpoint for the Ekk0 game/chat interface.
- `app/api/scorecard/ingest/route.ts`: Endpoint for ingesting scorecard metrics.
- `docs/`: Extensive documentation directory covering brand identity, research philosophy, and strategic planning.
- `next.config.mjs`: Next.js configuration with `standalone` output and Turbopack root definition.

## API / Exports
- **Research API**: Articles are served via `lib/research.ts` and rendered through the `[slug]` dynamic route.
- **Chat API**: `POST /api/ekk0/chat` and `POST /api/engineering/chat` handle conversational interactions.
- **Scorecard API**: `GET /api/scorecard/latest` and `POST /api/scorecard/ingest` manage platform performance metrics.
- **State API**: `GET /api/ekk0/state` provides current game/session state.

## Environment Variables
*Note: Check `.env.local` for local overrides.*
- `REDIS_URL`: Connection string for Upstash Redis.
- `GEMINI_API_KEY`: Authentication for the `@ekkos/gemini` package.
- `NEXT_PUBLIC_ANALYTICS_ID`: Vercel Analytics tracking.

## Conventions
- **App Router**: Uses the Next.js 16 App Router pattern with `(main)` route groups.
- **Styling**: Tailwind CSS with `clsx` and `tailwind-merge` for dynamic class composition.
- **Data Management**: Research content is decoupled from UI, stored in `lib/research.ts`.
- **Component Architecture**: UI components are modularized in `components/` and `app/engineering/`.
- **Build**: Uses `standalone` output mode for optimized Vercel deployment.
